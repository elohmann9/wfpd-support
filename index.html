Apparatus IT Support Console

Locate symptoms below to inspect recommended actions and escalation triggers.

CAD Tablet Offline

▼
iPad / WiFi
Tablet Command iPad shows red offline banner, and dispatch CAD logs fail to load.
Correction Action:
  • Verify iPad is on primary network: WFPD-[UNIT].
  • Never connect to the -GUEST network (guest isolation firewall blocks CAD sync routing).
  • Toggle iPad WiFi off/on to renew DHCP lease. Verify IP resolves to static reservation 10.7.X.50.
Escalate: If IP resolves to .50 but CAD remains red.

Starlink Offline

▼
Starlink
Rig has no internet; cell coverage is dead and Starlink status is offline in NetCloud.
Correction Action:
  • Confirm Starlink router power is active (check vehicle inverter switch and outlet plug).
  • Verify Bypass Mode is toggled on in the Starlink App.
  • Check ethernet adapter seats in both router and Cradlepoint WAN port.
Escalate: If hardware has power/link but fails to connect.

Starlink Stuck / Stowed

▼
Starlink
Starlink dish remains flat, refuses to rotate, or app reports status as "Stowed".
Correction Action:
  • Connect to Starlink router and open the Starlink App on your device.
  • Go to Settings, scroll down, and tap Unstow to deploy the dish motors.
  • Cycle the Starlink power supply to trigger a full motor reset and re-alignment loop.
Escalate: If dish motors refuse to engage.

Emergency USB Tether

▼
iPad Fallback
Rig primary cellular AND Starlink both fail; emergency internet link is required.
Correction Action:
  • Connect work iPad to Cradlepoint IBR1900 USB port using Apple USB cable.
  • On the iPad, open Settings -> Personal Hotspot and toggle it ON.
  • Cradlepoint will auto-detect the USB interface as an active WAN fallback connection.
Notice: Turn Personal Hotspot OFF when primary WAN returns.

Work Phone No Service

▼
iOS Cellular
Work iPhone or iPad has "No Service" or cellular data is unresponsive.
Correction Action:
  • Toggle Airplane Mode ON for 10 seconds, then OFF to force a clean re-registration on FirstNet.
  • Ensure Cellular Data Roaming is toggled ON under Cellular Settings.
  • Eject physical FirstNet SIM tray, inspect for dust/alignment, and re-seat.
Escalate: Wipes, eSIM updates, or carrier locks.

App Update Blocked

▼
MDM / Updates
Work app (Tablet Command, Maps) fails to update or gets stuck on "Waiting".
Correction Action:
  • MDM blocks large app downloads over cell data. Connect to station WiFi or high-speed rig WiFi.
  • Go to Settings -> General -> VPN & Device Management to verify the MDM profile is active.
  • Do not sign into personal Apple IDs; updates are managed automatically.
Escalate: Wipes, MDM token resets.

NCM Showing Offline

▼
NetCloud
All district apparatus show offline in NCM simultaneously.
Correction Action:
  • Visit status.cradlepoint.com to check system status.
  • Cradlepoint cloud portal maintenance regularly flags all routers offline even when vehicle internet is working fine.
Notice: Do not reset hardware. Wait.

Not Tracking (Versaterm)

▼
AVL / CAD
Dispatch reports "you are not tracking on CAD" or your unit is missing from dispatch maps.
Correction Action:
  • Verify the Cradlepoint router's GPS status light is solid green (on the IBR1900 hardware panel).
  • Check that the Tablet Command app has your correct vehicle unit ID active.
  • GPS tracking relies on ignition sense. Cycle vehicle engine off and back on to reboot the AVL tracking unit.
Escalate: Required if GPS light is off/flashing red.

Stuck Location (iPad Map)

▼
iPad GPS
iPad shows incident details, but the location cursor on the map is stuck or greyed out.
Correction Action:
  • Open iOS Settings -> Privacy & Security -> Location Services.
  • Ensure Location Services is enabled globally, and Tablet Command is set to Always Allow.
  • Ensure Precise Location is toggled ON under the Tablet Command location settings.
Notice: Metal bays block GPS signals. Test outdoors.

Need IT Spares / Protectors

▼
Hardware
Crew needs replacement screen protectors, chargers, vehicle mounts, or cables.
Correction Action:
  • Check the marked IT Supply Box located in the station office cabinet (Station 7 & Station 19).
  • Only use Apple-certified chargers from the box; generic cords cause screen noise and battery issues.
  • Clean the screen with an alcohol wipe from the box before applying new screen protectors.
Notice: Leave a note if you take the last spare.

Logins & WiFi Passwords

▼
Accounts
Locked out of dispatch apps, forgot account passwords, or need rig Wi-Fi security keys.
Correction Action:
  • Vaultwarden Manager: All shared credentials, logins, and WiFi security keys are stored in the district Vaultwarden vault.
  • Rig Wi-Fi Keys: Stored securely in Vaultwarden, or printed on the bottom of the Cradlepoint router bracket in the wiring bay.
  • App Lockouts: For Tablet Command or Target Solutions, reset using your standard woodsidefire.org Active Directory credentials.
Escalate: For Vaultwarden access lockouts.
